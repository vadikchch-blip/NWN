# First Access: инвайты из Excel и админ-страница бронирований

**Дата:** 2025-03-02

## Цель

- Загружать список гостей из Excel (файл «Список клиентов.xlsx»), создавать для каждого уникальный инвайт и ссылку.
- Дать админу отдельную страницу: список инвайтов (с копированием ссылок) и список всех бронирований.

## Подход

Отдельная страница `admin-first-access.html` (доступ только для администратора): два блока — **Инвайты** и **Бронирования**. Создание инвайтов — только скриптом из Excel; веб-интерфейс только для просмотра и копирования ссылок.

## Компоненты

### 1. Скрипт загрузки инвайтов

- **Файл:** `scripts/seed_first_access_invites.js`
- **Вход:** Excel (.xlsx). Путь по умолчанию: `Список клиентов.xlsx` в корне проекта. Переопределение: env `FIRST_ACCESS_INVITES_XLSX` или первый аргумент скрипта.
- **Колонки:** одна колонка «ФИО» (или «Имя», «Full name») либо две «Фамилия» и «Имя» — склейка в `full_name`.
- **Период доступа:** один для всех. Env: `FIRST_ACCESS_STARTS_AT` / `FIRST_ACCESS_ENDS_AT` (ISO) или аргументы `--from`, `--to`. Дефолт: сейчас и +7 дней.
- **Токен:** уникальный (nanoid или uuid), при коллизии — перегенерация.
- **Поведение:** только вставка новых записей в `first_access_invites`, без удаления и без дедупликации по ФИО.

### 2. API для админки (только role_id === 1)

- **GET `/api/first-access/admin/invites`**  
  Список инвайтов: id, full_name, token, access_starts_at, access_ends_at, is_active, created_at, invite_url.  
  `invite_url` = BASE_URL + `/access/supreme-first-access.html?token=` + token. BASE_URL из env или req.

- **GET `/api/first-access/admin/reservations`**  
  Все бронирования с джойнами: гость (full_name), товар (title, article), размер, статус, даты. Сортировка по дате резерва. Опционально query `status=active`.

### 3. Страница admin-first-access.html

- Доступ: проверка JWT и role_id === 1 (как в admin.html); иначе редирект.
- Секция **Инвайты:** таблица (ФИО, ссылка, активен до, статус), кнопка «Копировать» у ссылки.
- Секция **Бронирования:** таблица (гость, товар, размер, статус, дата).
- Стиль и хедер в духе существующей админки.

### 4. Безопасность

- Оба API защищены middleware `requireAdmin` (JWT + role_id === 1).
- Страница при загрузке вызывает `/api/auth/me` и при отсутствии админа перенаправляет на логин или главную.

## Ссылка-инвайт

Формат без изменений: `https://<site>/access/supreme-first-access.html?token=<token>`. Верификация и бронирование на стороне гостя не меняются.
